## Gotchas & Watch-outs

- **Trufflehog v3.95.2's legacy 40-hex GitHub PAT detector is keyword-gated by variable-name prefix.** Tokens stored under env vars without a GitHub keyword (`HYPERTERM_*`, `MY_APP_TOKEN`, `STASH_*`, etc.) are invisible to that detector — not flagged-and-unverified, but not even surfaced as candidates. Always pair trufflehog with the keyword-agnostic manual hex-env grep documented in the postmortem (`git log -p | rg '^\+.*=\s*["'\''][A-Fa-f0-9]{32,64}["'\'']'`). The new v2 detector catches modern prefixed tokens (`ghp_…`, `gho_…`) by the value's structural prefix — that's keyword-independent — but legacy 40-hex stays in the v1 detector with the keyword gate.
- **`Verified: false` from a secret scanner ≠ `revoked`.** Multiple paths produce that result: token actually revoked, OR token works but the verification probe hits an endpoint scope-blocked for the token. Cross-check at the platform UI directly (GitHub PAT page, etc.) before drawing conclusions.
- **Force-push on an open PR with /ultrareview history is fine** — it marks the line-anchored review threads as "outdated" but the comments stay visible. The dotfiles repo convention ("Force-push on draft PRs is OK when the branch is yours") extends here. The harness will challenge the operation as destructive — needs explicit `/permissions` approval per attempt.
- **Dangling commits on origin after force-push** stay reachable via direct-SHA URLs (`https://github.com/<owner>/<repo>/commit/<full-sha>`) for some retention window — hours to days for typical free-plan accounts, longer for orgs. The unredacted-PAT commit `9405805` is one of these now. Tokens are dead so practical risk is zero, but worth knowing the URL still resolves until GitHub's GC fires.
- **`.git/objects/` shows up in trufflehog filesystem scans.** Post-amend or post-rebase rescans will surface orphaned blobs (the previous version of any rewritten file) until `git gc --prune=now` runs locally. Don't mistake "trufflehog reports N working-tree findings after force-push amend" for "N current-tree leaks" — confirm with `git rev-list --all --objects | grep <blob-sha>` first.
- **`gh api /user/settings/billing/{actions,shared-storage}` returns 404 in 2026.** The billing-API endpoints have changed or moved. Web UI at https://github.com/settings/billing/summary is the working path for current usage.
- **Public-repo Actions minutes are unmetered.** Verified 2026-05-06 against https://docs.github.com/en/billing/concepts/product-billing/github-actions — "GitHub Actions usage is free for self-hosted runners and for public repositories that use standard GitHub-hosted runners." So no minute-cap risk on this repo regardless of macOS leg slowness. Storage IS metered (500 MB artifacts + 10 GB cache, shared with Packages including GHCR), so heavy `publish-ci-image.yml` history could eventually crowd that quota.
- **macOS CI variance is real** — this PR's macOS leg ran 15m6s vs the 9m40s baseline (~55% over). Not a code regression (PR is doc-only); just runner-pool jitter. Worth noting before chasing slow-test ghosts on future runs.

Reviewers reading the PR will see them marked outdated; the addressed-via-amend reply post explains the resolution.
